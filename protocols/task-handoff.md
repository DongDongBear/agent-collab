# 📨 Task Handoff Protocol v0.1

## 概述
定义 Agent 之间如何交接任务。

## 流程

```
发起方                              接收方
  │                                   │
  ├── 1. 评估任务，判断需要交接 ──────►│
  │      (能力不匹配 / 环境不匹配)     │
  │                                   │
  │◄── 2. 确认接收 / 拒绝 ────────────┤
  │      (能力检查 / 资源检查)         │
  │                                   │
  ├── 3. 传递上下文 ─────────────────►│
  │      (任务描述、相关文件、约束)     │
  │                                   │
  │◄── 4. 执行结果 / 进度更新 ────────┤
  │                                   │
  └── 5. 确认完成 ───────────────────►│
```

## 通信方式（按优先级）

1. **OpenClaw nodes.run** — 直接执行命令（最快）
2. **OpenClaw sessions_send** — 发送消息到对方 session
3. **Git commit** — 通过仓库同步（异步，适合非紧急）
4. **飞书消息** — 经 DongDong 中转（降级方案）

## 任务描述格式

```json
{
  "task_id": "unique-id",
  "from": "xiaoxiaodong",
  "to": "hetao",
  "type": "request",
  "priority": "normal",
  "description": "需要做什么",
  "context": "相关上下文",
  "deadline": null,
  "artifacts": []
}
```

## 能力路由规则

| 任务类型 | 推荐 Agent | 原因 |
|---------|-----------|------|
| 部署 Web 服务 | 🐿️ 小小动 | 云端常驻 |
| 操作本地文件 | 🌰 核桃 | 本地环境 |
| 定时监控 | 🐿️ 小小动 | 7×24 在线 |
| 浏览器截图 | 🌰 核桃 | GUI 环境 |
| 代码编写 | 任意 | 按上下文就近 |

## 扩展

新 Agent 加入时：
1. 在 `agents/registry.json` 注册
2. 声明 capabilities
3. 自动参与能力路由
